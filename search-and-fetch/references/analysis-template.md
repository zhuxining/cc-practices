# 五维文章分析模板

对提取的网页内容进行结构化分析。并非所有维度都适用于每种内容——根据内容类型选择性应用。

## 维度说明

### 1. 文本解构

**适用**：所有文章类内容
**关注点**：核心论点、支撑论据、逻辑关系
**输出**：3-5 行，梳理文章的论证结构

```markdown
### 文本解构
- 核心论点：[一句话概括作者的中心主张]
- 论据支撑：[列出主要支撑点及其类型——数据/案例/类比/权威引用]
- 逻辑链：[论点之间的推理路径]
```

### 2. 概念提炼

**适用**：所有文章类内容
**关注点**：关键概念的精确含义及相互关联
**输出**：3-5 行，提炼核心概念

```markdown
### 概念提炼
- [概念A]：[精确定义] — 与[概念B]的关系是[xxx]
- [概念C]：[精确定义] — 在文中的特定含义区别于常见理解
- 概念网络：[概念间的层级或因果关系简述]
```

### 3. 批判审视

**适用**：观点文章、评论、论文、辩论性内容
**不适用**：纯新闻报道、教程、数据页
**关注点**：论证强弱、逻辑漏洞、视角局限
**输出**：3-5 行，指出论证的薄弱环节

```markdown
### 批判审视
- 论证强度：[哪些论点有充分支撑，哪些证据不足]
- 逻辑漏洞：[是否存在偷换概念、以偏概全、假二分法等]
- 视角局限：[作者立场/利益可能带来的偏见，缺失的对立视角]
```

### 4. 思想深化

**适用**：思想性文章、学术论文、哲学/社科类内容
**不适用**：新闻、教程、产品页、技术文档
**关注点**：隐含前提、未明言假设、哲学基础
**输出**：3-5 行，揭示文章的深层思想基础

```markdown
### 思想深化
- 隐含前提：[作者未明确说明但依赖的假设]
- 思想根基：[这些观点背后的哲学/理论传统]
- 延伸思考：[如果改变某个前提，结论会如何变化]
```

### 5. 实践转化

**适用**：包含可操作洞见的所有内容（方法论、策略、技术、案例分析）
**不适用**：纯理论讨论、纯观点表达
**关注点**：可行洞见与具体应用方法
**输出**：3-5 行，提炼可落地的行动项

```markdown
### 实践转化
- 可直接应用的方法/工具：[xxx]
- 应用场景：[在什么条件下最有效]
- 注意事项：[使用时的前提条件或局限]
```

## 内容类型 → 维度选择速查

| 内容类型 | 文本解构 | 概念提炼 | 批判审视 | 思想深化 | 实践转化 |
|---------|:-------:|:-------:|:-------:|:-------:|:-------:|
| 深度文章/评论 | v | v | v | v | v |
| 学术论文 | v | v | v | v | v |
| 新闻报道 | v | v | - | - | v |
| 产品/发布页 | v | v | - | - | v |
| 技术教程 | v | v | - | - | v |
| 数据/参考页 | 跳过分析，直接呈现 ||||
| API 文档 | 跳过分析，直接呈现 ||||

## 输出要求

- 每个维度 3-5 行，简洁精准
- 保留原文中的对比表格和关键数据
- 使用中文输出（除非原文为英文且用户未指定语言）
- 如果某个维度确实不适用，跳过即可，不必勉强填充
